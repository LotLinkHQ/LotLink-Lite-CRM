RV Sales CRM Mobile App - Design Document
Overview
The RV Sales CRM is a specialized mobile application built for RV dealership sales teams to track incoming inventory and intelligently match units with waiting customers. The system automates the process of notifying salespeople when a customer's desired RV arrives at the dealership.
Core Problem Solved
When a customer walks into an RV dealership and doesn't find their ideal unit in stock, the salesperson needs a way to:
Capture detailed customer preferences (specific models, features, dimensions, amenities)
Track incoming RV inventory as units arrive at the dealership
Automatically match incoming units with waiting customers based on preferences
Notify the sales team when a match is found so they can contact the customer

Quick access to main features

Primary Content and Functionality
Leads Management
Create Lead: Capture customer preferences during dealership visit
View Leads: Browse all waiting customers
Edit Lead: Update customer preferences if they change their mind
Delete Lead: Remove customer from waitlist
Search/Filter: Find leads by name, preferences, or date added
Inventory Tracking
Log Unit Arrival: Record new RV units as they arrive
View Inventory: See all units in stock
Edit Unit: Update unit information if details change
Mark as Sold: Remove sold units from inventory
Search/Filter: Find units by model, features, or store
AI Matching System
Automatic Matching: When a new unit arrives, AI compares it against all waiting leads
Match Scoring: Calculates confidence score based on preference match
Notification: Alerts sales team of matches via:
In-app notification
Email notification (optional)
SMS notification (optional)
Match History: Track which customers were notified about which units
Notifications & Alerts
Match Alert: "New Match! 2025 Tiffin Allegro Bus 45OPP arrived - 3 customers waiting"
Action Required: Sales team can view matched customers and contact them
Completion Tracking: Mark when customer is contacted, unit is sold, or match is dismissed
Key User Flows
Flow 1: New Customer Arrives (Lead Creation)
Salesperson meets customer in dealership
Opens app → Leads screen → "Add New Lead"
Enters customer name, email, phone
Selects preferences:
Either specific model OR feature filters
Saves lead
Lead is now in the system waiting for matches
Flow 2: New Unit Arrives (Inventory Entry)
Unit arrives at dealership
Salesperson opens app → Inventory screen → "Add Unit"
Scans VIN or enters unit details
Fills in specifications (model, length, amenities, price, store)
Saves unit
AI automatically matches against all waiting leads
Sales team receives notification if matches found
Flow 3: Match Notification (Sales Team Action)
Sales team receives notification: "2025 Tiffin Allegro Bus 45OPP matched with 3 customers"
Opens app → Matches screen
Views matched customers and their details
Clicks "Contact Customer" → Opens phone/email
After contacting customer:
Mark as "Contacted"
If customer buys: Mark unit as "Sold"
If customer declines: Dismiss match
Match is archived
Flow 4: Search for Specific Unit
Customer calls asking about a specific model (e.g., "Do you have a 2025 Tiffin Allegro Bus 45OPP?")
Salesperson opens app → Inventory screen → Search "2025 Tiffin Allegro Bus 45OPP"
Sees if unit is in stock
If in stock: Can view details and show customer
If not in stock: Can check if any leads are waiting for this model

Technical Considerations
Local-first: Store leads and inventory locally on device for offline access
Backend sync (optional): Sync data across multiple devices/stores if needed
AI Matching: Use backend LLM to intelligently match units with customer preferences
Notifications: In-app notifications immediately; email/SMS via backend service
Search: Fast local search with filters for leads and inventory
Performance: Optimize for quick data entry and retrieval (salespeople need speed)
